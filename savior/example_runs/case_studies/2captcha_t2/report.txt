[Vulnerability Overview]
================================================================================
ID              : T2
Name            : Unaligned First-party Verification Hijacking
Risk Level      : High

Description:
OAuth login can bypass email verification and link to unverified accounts.
When a user registers with email (requiring verification) but doesn't verify,
an attacker can use OAuth with the same email to activate and take over the account.

================================================================================
[Attack Analysis (Site-Specific)]
================================================================================
Attack Path:
1. Registered via email/password with test_account@example.com - system required email verification (confirmation code page)
2. In a new browser session, registered via Google OAuth with the same email - system created account without email verification
3. Logged in via email/password - accessed the exact same account created by OAuth (identical API Key confirmed)
4. OAuth and email registration linked to the same account, bypassing verification
Missing Security Check: OAuth registration should not auto-link to a pending unverified email registration, or should require the email owner to verify before merging accounts
Proof of Vulnerability: API Key from OAuth login and email/password login are identical, confirming same account access from both registration paths
Account Identifiers Compared: API Key=[REDACTED_SAME_API_KEY]

[General Remediation]
================================================================================
- Do not link OAuth accounts to unverified email registrations
- Require email verification before allowing any login method
- Treat OAuth registration and email registration as separate accounts until verified
- Implement proper account merging only after both methods verify email ownership
